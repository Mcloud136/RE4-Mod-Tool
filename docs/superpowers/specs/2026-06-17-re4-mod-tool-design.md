# RE4 Mod Tool - 服装mod颜色自定义自动化工具

## 项目概述

**RE4 Mod Tool** 是一款面向生化危机4重置版的专业级服装mod颜色自定义工具。工具提供从游戏贴图格式自动解析、直观颜色编辑、高质量3D实时预览到自动打包回游戏格式的完整工作流程。

**目标用户**：专业设计师，需要高级图像编辑功能和精确颜色控制的mod制作者。

**技术栈**：C++ 17 / Qt 6.8 / DirectX 11/12

**开发周期**：6-8个月（一次性开发全部功能）

### 1.2 模块间通信

使用Qt信号槽机制实现模块间通信，各核心模块独立运行，通过明确定义的信号和槽进行数据交换：

- **格式解析模块** 发出 `fileLoaded` 信号 → **颜色编辑模块** 接收并初始化编辑器
- **颜色编辑模块** 发出 `colorChanged` 信号 → **3D渲染模块** 接收并实时更新预览
- **路径配置模块** 发出 `texturePathChanged` 信号 → **格式解析模块** 接收并加载新贴图
- **批量处理模块** 发出 `taskProgressChanged` 信号 → **UI层** 接收并更新进度条

## 2. 格式转换模块

### 2.1 设计目标

实现游戏贴图格式（.tex/.dds）与可编辑图片格式（TGA/DDS/PNG）之间的双向转换，无需依赖第三方工具。

### 2.2 支持的格式转换

| 输入格式 | 输出格式 | 说明 |
|---------|---------|------|
| .tex | TGA | 无压缩，质量最高 |
| .tex | DDS | 保留压缩格式，文件小 |
| .tex | PNG | 通用格式，便于分享 |
| TGA | .tex | 转换回游戏格式 |
| DDS | .tex | 保留压缩信息 |
| PNG | .tex | 需要重新压缩 |

### 2.3 格式解析策略

**技术路线**：参考 [REE-Content-Editor](https://github.com/kagenocookie/REE-Content-Editor)（MIT许可）的开源实现，分析其.tex和.dds的解析逻辑作为参考。结合DirectXTex库的BC压缩编解码能力实现格式转换。

具体步骤：

1. **文件头解析**：读取并验证文件头magic number、版本号、尺寸、格式标记等元数据
2. **元数据提取**：解析Mipmap层级数、纹理数组大小、压缩格式标识
3. **图像数据提取**：根据元数据定位并提取原始像素数据
4. **BC压缩处理**：使用DirectXTex库进行BC1/BC3/BC5/BC7等格式的编解码
5. **Mipmap处理**：支持多级Mipmap的读取和生成

### 2.4 质量保证

- **无损转换**：在可能的情况下保持原始数据完全一致
- **格式验证**：转换前后验证文件完整性（校验和、尺寸一致性）
- **回退机制**：转换失败时保留原始文件不被覆盖
- **详细日志**：记录每次转换的参数、耗时、结果

## 3. 调色盘界面

### 3.1 设计目标

提供专业级的颜色编辑界面，支持多种颜色调整方式，所有修改实时反馈到预览中。

### 3.2 核心功能

#### 3.2.1 RGB滑块编辑器
- 四个独立滑块（R/G/B/A），范围0-255
- 支持直接输入数值
- 通道锁定功能（锁定某个通道不被修改）
- 滑块背景显示颜色渐变

#### 3.2.2 HSL滑块编辑器
- 圆形色相选择器（0-360度）
- 线性饱和度滑块（0-100%）
- 线性亮度滑块（0-100%）

#### 3.2.3 通道分离显示
- 单独查看R/G/B/A通道的灰度图
- 单独编辑某个通道
- 通道混合比例调整

#### 3.2.4 预设颜色系统
- 内置常用颜色色板
- 用户自定义收藏夹
- 颜色配置文件导入/导出

#### 3.2.5 取色器工具
- 从当前贴图中取色（贴图内取色）
- 从屏幕任意位置取色（屏幕取色）
- 记录最近使用的颜色历史

#### 3.2.6 高级调整
- 亮度/对比度：整体明暗和对比调整
- 饱和度：颜色鲜艳程度调整
- 色相/色调：整体色调偏移
- 曲线调整：高级色调映射（可选功能）

### 3.3 交互设计原则

1. **实时反馈**：所有调整立即在预览中显示
2. **撤销/重做**：支持无限步撤销/重做（基于命令模式）
3. **快捷键**：常用操作支持快捷键
4. **工具提示**：每个控件都有详细说明
5. **响应式布局**：支持窗口大小调整

## 4. 3D实时预览系统

### 4.1 设计目标

内置基于DirectX 11/12的高质量3D渲染引擎，提供完整的模型预览能力，颜色修改实时反映到3D模型上。

### 4.2 核心功能

#### 4.2.1 DirectX渲染引擎
- 完整的3D渲染管线（顶点处理、光栅化、像素着色）
- PBR材质渲染支持
- 多光源系统（环境光、方向光、点光源）
- 实时阴影渲染

#### 4.2.2 模型加载
- **阶段1（MVP）**：支持标准3D格式（FBX/glTF）作为预览模型来源，用户可手动加载角色模型
- **阶段2（完整版）**：参考REE-Content-Editor的REE-Lib库实现游戏.mesh格式的直接加载和解析
- 自动加载关联的材质和贴图
- 基础骨骼动画预览
- 多细节层次（LOD）渲染

#### 4.2.3 视角控制
- **旋转**：鼠标左键拖拽旋转
- **缩放**：鼠标滚轮缩放
- **平移**：鼠标右键拖拽平移
- **预设视角**：正面、侧面、背面、顶部等快捷切换

#### 4.2.4 光照设置
- 环境光：强度和颜色可调
- 方向光：角度、强度、颜色可调
- 点光源：位置、强度、衰减可调
- 光照预设：日光、夜景、工作室等预设方案

#### 4.2.5 显示选项
- 线框模式：显示模型线框
- 法线显示：可视化顶点法线方向
- UV显示：可视化UV坐标
- 材质/光照切换：不同渲染模式切换

#### 4.2.6 实时同步
- 调色盘中的颜色修改实时反映到3D预览
- 贴图修改后自动更新3D模型上的纹理
- 脏区域检测，只更新修改部分，避免全量重绘

### 4.3 性能优化

1. **异步渲染**：在后台线程渲染，不阻塞UI线程
2. **脏区域检测**：仅重新渲染修改过的部分
3. **LOD切换**：根据相机距离自动切换模型细节层次
4. **纹理压缩**：使用GPU友好的纹理格式
5. **批量渲染**：合并渲染调用，减少API开销

## 5. 路径配置系统

### 5.1 设计目标

支持多Mod、多部位的贴图路径灵活配置，提供模板系统和快速访问功能。

### 5.2 核心功能

#### 5.2.1 Mod配置管理
- Mod列表：管理多个Mod的配置（添加、删除、编辑）
- 导入/导出：支持JSON格式的配置文件导入导出，便于分享
- 快速切换：在不同Mod间快速切换

#### 5.2.2 贴图路径配置
- 部件管理：为每个Mod配置不同的部件（身体、头部、手套、鞋子等）
- 路径设置：为每个部件设置独立的贴图文件路径
- 自动扫描：自动扫描目录中的贴图文件并识别
- 手动浏览：通过文件对话框手动选择贴图文件

#### 5.2.3 路径模板系统

支持变量替换的路径模板：

| 变量 | 说明 | 示例 |
|------|------|------|
| {角色} | 角色名称 | Leon, Ashley, Ada |
| {服装} | 服装名称 | Default, Costume01, DLC |
| {部件} | 部件名称 | Body, Head, Glove, Shoe |
| {贴图类型} | 贴图类型 | _d (diffuse), _n (normal), _s (specular) |
| {分辨率} | 贴图分辨率 | 1024, 2048, 4096 |

模板示例：`native\STM\character\{角色}\{服装}\{部件}_d.tex`

#### 5.2.4 快速访问
- 最近使用：记录最近使用的Mod和路径
- 收藏夹：收藏常用的路径和配置

### 5.3 配置文件格式

使用JSON格式存储配置：

```json
{
  "mods": [
    {
      "name": "Leon默认服装",
      "basePath": "C:\\RE4\\Mods\\Leon\\Default",
      "textures": [
        {
          "partName": "身体",
          "texturePath": "native\\STM\\character\\Leon\\Body\\body_d.tex",
          "textureType": "diffuse"
        }
      ]
    }
  ],
  "templates": [
    {
      "name": "角色服装模板",
      "basePath": "native\\STM\\character\\{角色}\\{服装}\\",
      "texturePattern": "{部件}_d.tex"
    }
  ],
  "recentMods": [],
  "recentPaths": []
}
```

## 8. 数据流和系统集成

### 8.1 核心数据流

#### 贴图加载流程
```
用户打开文件 → 文件I/O读取 → 格式解析 → 图像数据提取 →
颜色空间转换 → 缓存原始数据 → 更新UI预览 → 加载到3D引擎 → 显示3D预览
```

#### 颜色编辑流程
```
用户调整颜色 → 颜色参数计算 → 实时图像处理 → 更新UI预览 →
更新3D预览 → 记录历史操作 → 更新撤销/重做栈
```

#### 贴图保存流程
```
用户保存文件 → 获取当前图像数据 → 颜色空间逆转换 → 格式压缩 →
文件I/O写入 → 验证文件完整性 → 更新缓存 → 通知用户完成
```

#### 批量处理流程
```
用户配置批量任务 → 任务队列创建 → 多线程处理 → 逐个任务执行 →
进度更新 → 错误处理 → 结果汇总 → 通知用户完成
```

### 8.2 缓存策略

#### 内存缓存
- LRU策略：最近最少使用的数据优先淘汰
- 可配置的缓存上限（默认100MB）
- 缓存命中率统计

#### 文件缓存
- 转换后的文件缓存到临时目录
- 缓存索引管理
- 自动清理过期缓存（默认30天）

### 8.3 日志系统

| 日志级别 | 用途 |
|---------|------|
| Debug | 开发调试信息 |
| Info | 一般运行信息 |
| Warning | 警告信息（可恢复的问题） |
| Error | 错误信息（操作失败） |
| Fatal | 致命错误（程序无法继续） |

日志支持：
- 控制台输出
- 文件输出（带自动轮转）
- 日志级别过滤
- 按分类过滤

## 9. 错误处理和测试策略

### 9.1 错误分类和处理

| 错误类型 | 严重程度 | 处理策略 | 恢复方式 |
|---------|---------|---------|---------|
| 文件不存在 | 中 | 跳过该文件，继续处理 | 选择其他文件 |
| 格式不支持 | 中 | 跳过该文件，记录日志 | 转换格式 |
| 内存不足 | 高 | 释放缓存，重试 | 减少处理量 |
| 渲染失败 | 中 | 降级渲染，继续 | 切换渲染模式 |
| 配置损坏 | 中 | 使用默认配置 | 重置配置 |
| 数据损坏 | 高 | 从备份恢复 | 恢复备份 |

## 附录A：已知限制和约束

1. **平台限制**：仅支持Windows平台（DirectX依赖）
2. **游戏版本**：主要针对生化危机4重置版，其他RE Engine游戏可能部分兼容
3. **格式支持**：.tex格式解析基于逆向工程，可能不覆盖所有变体
4. **3D模型**：仅支持静态模型预览，动画支持有限
5. **在线功能**：预设在线分享为可选功能，需网络连接

## 附录B：未来扩展方向

1. **游戏内集成**：探索通过mod框架实现游戏内直接访问自定义颜色
2. **更多游戏支持**：扩展到其他RE Engine游戏（生化危机2/3重置版等）
3. **AI辅助**：智能颜色建议、风格迁移
4. **社区功能**：在线预设库、作品分享平台
5. **插件系统**：支持第三方插件扩展功能
