## 1. 简介

### 1.1 什么是RE4 Mod Tool

RE4 Mod Tool 是一款专为《生化危机4重置版》设计的服装mod颜色自定义工具。它提供了：

- **自动格式转换**：无需第三方软件，直接解析和生成游戏贴图格式
- **直观的调色盘界面**：RGB滑块、颜色调整、预设系统
- **实时3D预览**：通过REFramework在游戏中实时预览效果
- **批量处理**：同时处理多个贴图文件
- **Mod路径管理**：支持多mod多部件的灵活配置

### 1.2 支持的格式

| 格式 | 读取 | 写入 | 说明 |
|------|------|------|------|
| .tex | ✅ | ✅ | RE Engine 贴图格式 |
| .dds | ✅ | ✅ | DirectDraw Surface |
| .png | ✅ | ✅ | 便携式网络图形 |
| .tga | ✅ | ✅ | Targa 格式 |

### 1.3 支持的压缩格式

| 格式 | 说明 |
|------|------|
| BC1 | DXT1 - 无alpha或1位alpha |
| BC2 | DXT3 - 显式alpha |
| BC3 | DXT5 - 插值alpha |
| BC4 | 单通道 |
| BC5 | 双通道（法线贴图） |
| BC7 | 高质量RGBA |

## 3. 安装指南

### 3.1 安装桌面工具

#### 方法1：使用安装程序（推荐）

1. 下载 `RE4ModTool_Setup_1.0.0.exe`
2. 双击运行安装程序
3. 按照向导完成安装
4. 选择安装目录（默认：`C:\Program Files\RE4 Mod Tool`）
5. 选择是否创建桌面快捷方式
6. 完成安装

#### 方法2：便携版

1. 下载 `RE4ModTool_v1.0.0.zip`
2. 解压到任意目录
3. 运行 `RE4ModTool.exe`

### 3.2 安装REFramework（游戏内预览）

1. 访问 [REFramework Releases](https://github.com/praydog/REFramework/releases)
2. 下载最新版本
3. 解压 `dinput8.dll` 到 RE4 游戏目录
   ```
   Steam\steamapps\common\RESIDENT EVIL 4\
   ```
4. 启动游戏验证REFramework工作正常（应显示REFramework覆盖层）

### 3.3 安装DLL插件（游戏内预览）

#### 方法1：使用安装程序

1. 运行 RE4 Mod Tool 安装程序
2. 在组件选择页面，勾选 "REFramework DLL插件"
3. 选择REFramework插件目录：
   ```
   Steam\steamapps\common\RESIDENT EVIL 4\reframework\plugins\
   ```
4. 完成安装

#### 方法2：手动安装

1. 编译DLL插件：
   ```bash
   cd re4modtool_dll
   cmake -B build -G "Visual Studio 17 2022" -A x64
   cmake --build build --config Release
   ```
2. 复制 `re4modtool.dll` 到：
   ```
   Steam\steamapps\common\RESIDENT EVIL 4\reframework\plugins\
   ```

## 4. 快速入门

### 4.1 第一次使用

1. **启动工具**
   - 双击桌面快捷方式
   - 或从开始菜单启动 "RE4 Mod Tool"

2. **打开贴图文件**
   ```
   文件 → 打开 (Ctrl+O)
   ```
   - 浏览到游戏贴图目录
   - 选择 `.tex`、`.dds`、`.png` 或 `.tga` 文件
   - 点击"打开"

3. **查看贴图**
   - 中央区域显示贴图预览
   - 鼠标滚轮：缩放
   - 鼠标中键：平移
   - 鼠标左键：拾取颜色

4. **编辑颜色**
   - 使用左侧 RGB 滑块调整颜色
   - 或使用右侧颜色调整面板
   - 实时预览效果

5. **保存文件**
   ```
   文件 → 保存 (Ctrl+S)
   ```
   - 保存到原文件
   - 或使用"另存为"保存到新位置

### 4.2 基本工作流程

```
打开贴图 → 预览 → 编辑颜色 → 预览效果 → 保存
```

## 6. 游戏内预览

### 6.1 前置条件

1. 已安装 REFramework
2. 已安装 DLL 插件
3. 游戏已启动

### 6.2 连接流程

1. 启动 RE4 游戏
2. REFramework 自动加载
3. DLL 插件自动加载
4. 启动 RE4 Mod Tool
5. 等待连接（通常 1-2 秒）
6. 状态栏显示 "游戏：已连接"

### 6.3 游戏内 ImGui 窗口

游戏内会显示一个 ImGui 窗口，包含：

#### 颜色面板
- **R/G/B/A 滑块**：快速调整颜色
- **颜色预览**：显示当前颜色
- **应用按钮**：应用颜色到游戏贴图

#### 相机面板
- **预设下拉框**：选择相机预设
  - 正面 (Front)
  - 背面 (Back)
  - 左侧 (Left)
  - 右侧 (Right)
  - 顶部 (Top)
  - 底部 (Bottom)
  - 自由 (Free)
- **距离滑块**：调整相机距离
- **FOV滑块**：调整视场角
- **应用预设按钮**：应用相机预设

#### 连接面板
- **状态指示**：绿色/红色显示连接状态
- **连接/断开按钮**：手动控制连接
- **发送Ping按钮**：测试连接

### 6.4 实时预览流程

```
1. 在桌面工具中打开贴图
2. 调整颜色（RGB滑块/颜色调整/预设）
3. 游戏内实时显示效果
4. 旋转相机查看不同角度
5. 满意后保存文件
6. 游戏内自动热替换贴图
```

### 6.5 F1 快捷键

- 按 F1 键显示/隐藏游戏内 ImGui 窗口

## 7. 批量处理

### 7.1 批量格式转换

#### 使用方法

1. 打开批量转换对话框：
   ```
   工具 → 批量转换...
   ```

2. 选择输入文件：
   - 点击"添加文件"选择单个文件
   - 或点击"添加文件夹"选择整个文件夹

3. 选择输出目录：
   - 点击"浏览"选择输出目录

4. 选择输出格式：
   - PNG - 通用格式，便于分享
   - TGA - 无压缩，质量最高
   - DDS - 保留压缩格式
   - TEX - 游戏格式

5. 点击"确定"开始转换

6. 等待处理完成，进度条显示当前进度

#### 注意事项

- 批量转换会覆盖同名文件（如果选择相同目录）
- 大文件处理需要较长时间
- 可以点击"取消"中断处理

### 7.2 批量颜色调整

#### 使用方法

1. 打开批量调色对话框：
   ```
   工具 → 批量调色...
   ```

2. 选择输入文件

3. 选择输出目录

4. 设置调整参数：
   - **亮度**：-255 到 +255
   - **对比度**：0.0 到 3.0（1.0 = 不变）
   - **饱和度**：0.0 到 3.0（1.0 = 不变）
   - **色相偏移**：-180 到 +180 度

5. 点击"确定"开始处理

### 7.3 批量尺寸调整

#### 使用方法

1. 打开批量调整尺寸对话框：
   ```
   工具 → 批量调整尺寸...
   ```

2. 选择输入文件

3. 选择输出目录

4. 设置目标尺寸：
   - **宽度**：目标宽度（像素）
   - **高度**：目标高度（像素）

5. 点击"确定"开始处理

#### 注意事项

- 尺寸调整会改变贴图比例（如果宽高比不匹配）
- 建议保持2的幂次尺寸（如 512、1024、2048、4096）

## 8. Mod路径管理

### 8.1 添加Mod

1. 在左下角"Mod路径"面板
2. 点击"添加Mod"
3. 填写信息：
   - **Mod名称**：如 "Leon默认服装"
   - **基础路径**：Mod的根目录
   - **部件列表**：如 "身体,头部,手套,鞋子"

4. 点击"确定"保存

### 8.2 编辑Mod

1. 在Mod列表中选择要编辑的Mod
2. 点击"编辑Mod"
3. 修改信息
4. 点击"确定"保存

### 8.3 删除Mod

1. 在Mod列表中选择要删除的Mod
2. 点击"删除Mod"
3. 确认删除

### 8.4 路径模板

#### 变量说明

| 变量 | 说明 | 示例 |
|------|------|------|
| `{角色}` | 角色名称 | Leon, Ashley, Ada |
| `{服装}` | 服装名称 | Default, Costume01, DLC |
| `{部件}` | 部件名称 | Body, Head, Glove, Shoe |
| `{贴图类型}` | 贴图类型 | _d (diffuse), _n (normal), _s (specular) |

#### 模板示例

```
native\STM\character\{角色}\{服装}\{部件}_d.tex
```

展开为：
```
native\STM\character\Leon\Default\body_d.tex
```

## 9. 预设系统

### 9.1 使用预设

1. 在右下角"预设"面板
2. 从列表中选择预设
3. 点击"应用"按钮
4. 颜色调整自动应用到当前贴图

### 9.2 保存预设

1. 调整颜色到满意状态
2. 点击"保存当前"
3. 输入预设名称和描述
4. 点击"确定"保存

### 9.3 删除预设

1. 在预设列表中选择要删除的预设
2. 点击"删除"
3. 确认删除

### 9.4 导出/导入预设

#### 导出预设

1. 选择要导出的预设
2. 点击"导出"
3. 选择保存位置
4. 保存为 `.json` 文件

#### 导入预设

1. 点击"导入"
2. 选择 `.json` 预设文件
3. 预设自动添加到列表

### 9.5 预设内容

每个预设包含：
- 名称和描述
- 颜色参数（主色调、强调色）
- 调整参数（亮度、对比度、饱和度、色相）
- 应用规则（是否保留alpha、混合模式）

## 11. 常见问题

### 11.1 安装问题

**Q: 安装程序提示"缺少Qt运行库"？**
A: 请确保安装了完整的Qt运行库，或使用安装程序版本（已包含所有依赖）。

**Q: 安装后无法启动？**
A: 检查是否安装了 Visual C++ Redistributable 2022。

### 11.2 格式转换问题

**Q: 无法打开 .tex 文件？**
A: 确保文件是有效的 RE Engine 贴图格式。损坏的文件无法打开。

**Q: 转换后贴图质量下降？**
A: 
- 使用 TGA 格式保持最高质量
- 避免多次转换（每次转换可能损失质量）
- BC7 压缩会有轻微质量损失（正常现象）

**Q: 转换后文件大小变化很大？**
A: 不同格式的压缩方式不同：
- PNG：无损压缩，文件较小
- TGA：无压缩，文件较大
- DDS：BC压缩，文件最小

### 11.3 颜色编辑问题

**Q: 颜色调整没有效果？**
A: 
- 确保点击了"应用"或"全部应用"按钮
- 检查是否选中了正确的通道
- 尝试重置后重新调整

**Q: 撤销/重做不工作？**
A: 
- 确保有可撤销的操作
- 检查撤销栈是否已满（最多50步）

### 11.4 游戏内预览问题

**Q: 游戏内没有显示ImGui窗口？**
A: 
1. 检查REFramework是否正确安装
2. 检查DLL插件是否在正确目录
3. 查看游戏目录的 `reframework.log` 日志
4. 按 F1 尝试显示/隐藏窗口

**Q: 桌面工具显示"游戏：未连接"？**
A: 
1. 确保游戏已启动
2. 确保REFramework正常工作
3. 检查防火墙是否阻止连接
4. 重启游戏和桌面工具

**Q: 修改颜色后游戏内没有变化？**
A: 
1. 确保DLL插件已正确安装
2. 确保桌面工具显示"游戏：已连接"
3. 点击ImGui窗口中的"应用"按钮
4. 检查贴图路径是否正确

### 11.5 批量处理问题

**Q: 批量处理很慢？**
A: 
- 批量处理使用多线程，大文件需要时间
- 进度条显示当前进度
- 可以点击"取消"中断处理

**Q: 批量处理部分文件失败？**
A: 
- 检查失败文件是否损坏
- 确保输出目录有写入权限
- 查看错误日志了解具体原因

## 12. 故障排除

### 12.1 日志文件

日志文件位置：
```
%APPDATA%\RE4ModTool\re4modtool.log
```

查看日志可以帮助诊断问题。

### 12.2 常见错误代码

| 错误 | 说明 | 解决方案 |
|------|------|---------|
| 文件不存在 | 指定的文件路径不存在 | 检查文件路径 |
| 格式不支持 | 不支持的文件格式 | 使用支持的格式 |
| 内存不足 | 系统内存不足 | 关闭其他程序 |
| 权限不足 | 没有写入权限 | 以管理员身份运行 |
| 格式损坏 | 文件格式损坏 | 使用原始文件 |

### 12.3 性能优化

**Q: 工具运行缓慢？**
A: 
1. 关闭不必要的程序
2. 使用SSD存储贴图文件
3. 减少同时打开的贴图数量
4. 降低3D预览的分辨率

**Q: 内存占用过高？**
A: 
1. 关闭不需要的预览窗口
2. 减少撤销步数
3. 定期保存并重启工具
